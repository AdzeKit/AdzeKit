// adzekit

Prehistoric tools. Modern brains.

AdzeKit is a markdown-native productivity workbench. Everything lives in plain text files you own, edit with any tool, and version with git. No databases, no proprietary formats, no lock-in.

The name comes from the adze -- one of the oldest woodworking tools. Simple, durable, effective. The shed metaphor runs through the entire system: your workspace is a shed, your files are the backbone, raw materials sit in stock, and the bench is where you evaluate agent proposals.

Five Principles

1
Cap Work-in-Progress Max 3 active projects and 5 daily focus tasks. If you want to start something new, finish or defer something first.
2
Close Every Loop Every commitment to another person gets a response within 24 hours. A loop stays open until explicitly closed with evidence.
3
Protect Deep Work Batch communications. Minimize context switches. Guard uninterrupted blocks for real thinking.
4
Human Always Decides Agents draft and suggest. You approve. The backbone is your domain -- AI writes proposals to drafts/, never to the backbone directly.
5
Boring Technology Markdown, git, the filesystem. No framework churn, no migration debt. The backbone spec outlasts every tool built on top of it.

The Backbone

The backbone is the set of plain markdown files that hold your real data. Any directory that conforms to the backbone spec is an AdzeKit-compatible shed.

daily/ One file per day. Intentions, log, reflection.
loops/ Commitments: active, backlog, archive.
projects/ Active, backlog, archive. Context + log + notes.
knowledge/ Evergreen notes. One concept per file.
reviews/ Weekly review snapshots (ISO week).
bench.md Triage queue for agent proposals.
stock/ Raw material: transcripts, PDFs, exports.
drafts/ Agent-written proposals awaiting review.

Access Zones

The shed is split into two zones that enforce human control:

Zone Directories Who Writes
Backbone daily/ loops/ projects/ knowledge/ reviews/ bench.md Human only
Workbench stock/ drafts/ Agent-writable

Lifecycle Pattern

Both loops and projects follow the same three-state lifecycle:

  • Active -- currently in progress, top of mind
  • Backlog -- committed but not urgent, future work
  • Archive -- completed, preserved for reference

Move items between states by moving the file (projects) or cutting/pasting lines between files (loops). The filesystem is the state machine.

Key Commands

  • adzekit init -- create a new shed
  • adzekit today -- open or create today's daily note
  • adzekit sweep -- move [x] loops from active to archive
  • adzekit cull -- surface new drafts on the bench
  • adzekit status -- shed health at a glance
  • adzekit serve -- start this web UI
  • adzekit export -- convert markdown to .docx
